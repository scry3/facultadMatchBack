-index.html

Página principal o bienvenida.
Puede tener botón de “Registrarse” y “Iniciar sesión”.



-login.html

Formulario para login:

username

password
Hace un fetch al backend.




-register.html

Formulario para registrar usuario:

username

password

nombre ficticio

edad

carrera

descripción

instagram



-explorar.html

Pantalla donde el usuario ve otros perfiles y puede:

like

skip

Fetch a /users y /like/:id.




-match.html

Muestra los matches del usuario actual.
Cada match muestra el instagram del otro.



-estilos.css

Estilos globales del frontend.




-main.js

Código JS del front que maneja:

guardar sesión del usuario (si hace falta)

hacer los fetch al backend

cargar datos en las pantallas

-routes/auth.js

Define los endpoints relacionados al login y al registro, por ejemplo:

POST /register

POST /login

POST /logout

Solo define rutas, no la lógica.



-routes/users.js

Endpoints relacionados a usuarios, por ejemplo:

GET /me (trae datos del usuario logueado)

GET /users (trae lista de usuarios a mostrar en explorar)


-routes/matches.js

Rutas para ver los matches:

GET /matches (lista de matches del usuario actual)



-db/database.js

Acá se configura la conexión a SQLite y se crean las tablas si no existen.

Ej: tabla users, likes, matches.



-controllers/authController.js

Acá va la lógica del registro, login y logout:

validar datos

hashear password

guardar en base

crear la sesión




-controllers/userController.js

Lógica para:

devolver el perfil del usuario

devolver lista de usuarios (para el explorador)



-controllers/likeController.js

Lógica para manejar likes:

guardar un like

verificar si hay match



-controllers/matchController.js

Lógica para:

devolver la lista de matches

devolver el Instagram del match



-middlewares/authMiddleware.js

Middleware que protege rutas:

verifica si la sesión está activa

si no, bloquea el endpoint

Ej: no puede dar like si no está logueado.



-app.js

Une rutas, middlewares y configuración general de Express.


-server.js

Arranca el servidor (app.listen).


-package.json

Lista dependencias del backend (express, sqlite3, bcrypt, express-session, etc.)


-.env

Variables privadas:

SESSION_SECRET

config de DB (si fuera otro motor)
